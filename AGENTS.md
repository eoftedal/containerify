# AGENTS.md

Instructions for AI coding agents working in this repository.

## Scope And Source Of Truth

- Keep this file minimal and action-oriented.
- Link to existing docs instead of duplicating them:
  - [CLAUDE.md](CLAUDE.md) for architecture, pipeline flow, and conventions
  - [README.md](README.md) for CLI usage and examples
  - [CHANGELOG.md](CHANGELOG.md) for release history

## Fast Start

- Install deps: `npm install`
- Build before running tests that execute the CLI: `npm run build`
- Main quality gate: `npm run check`
- Main CI-style test: `npm run integrationTest`
- Docker-required local registry tests: `npm run registryTest`

## Critical Project Conventions

- The runtime CLI is built output in [lib/cli.js](lib/cli.js), not TypeScript source.
- Do not manually edit [src/version.ts](src/version.ts); it is generated by `npm run build` (`prebuild`).
- Use tabs for indentation and keep formatting aligned with existing style.
- Do not expand scope to unrelated files; keep edits minimal and targeted.

## CLI Safety Rules (Mandatory)

- Always validate command-line arguments thoroughly.
- In [src/cli.ts](src/cli.ts), enforce invalid/missing/conflicting options with explicit checks and clear error messages.
- Keep argument validation behavior deterministic and fail fast.
- For any new flag or option:
  - add parsing + validation in [src/cli.ts](src/cli.ts)
  - ensure interactions with existing flags are validated (mutual exclusivity, dependencies, required combinations)
  - update help text and documentation in [README.md](README.md)
  - add or update integration coverage in [tests/integration/test.sh](tests/integration/test.sh)

## Test Expectations For Changes

- If CLI behavior changes, run at least:
  - `npm run build`
  - `npm run integrationTest`
- If changes affect registry push/pull behavior and Docker is available, also run:
  - `npm run registryTest`

## Known Pitfalls

- On macOS, bash may be 3.x; avoid bash features like `mapfile` in test scripts.
- Integration tests are shell-based and compare output artifacts for reproducibility; avoid nondeterministic output unless intentional.

## Suggested Next Customizations

- Create a focused instruction file for CLI work (for example, `.github/instructions/cli.instructions.md`) to enforce argument-validation checklists on edits under `src/cli.ts`.
- Create a reusable prompt (for example, `.github/prompts/add-cli-flag.prompt.md`) that scaffolds all required changes for new flags (parsing, validation, README, changelog, tests).
- Use `/chronicle improve` periodically to mine session friction and refine these instructions.
